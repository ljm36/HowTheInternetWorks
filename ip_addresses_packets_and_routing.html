IP Addresses, Packets, and Routing

Each device on the planet has a unique IP address to communicate with each other. IP means Internet protocol, which lets one computer send a packet to another one knowing its IP address. Just like a home address, IP address have a certain pattern to the way that they are read. IP addresses are 32 bits long. The first section of the IP address represents the country/network, the second part represents the country/network, the fourth part represents subnetwork and the last part the device itself. This format is known as IPv4 and can handle up to 4 billion IP addresses, since the internet has grown so much more, we are transitioning slowly into IPv6 which has 128 bits per address which is the equivalent of 340 undecillion addresses.

Information goes to one computer to another, but instead of going directly to the receiver it will move across using different paths until it reaches its destination, it does not follow a fix path. A bigger packet will get broken up into smaller packets and get sent through different paths, but at the end once each packet reaches its destination it will assemble and combine into the main big packet. Routers makes the path for each packet to be delivered. TCP means Transmission Control Protocol, what this does is manages the sending and receiving of all your data as packets.
